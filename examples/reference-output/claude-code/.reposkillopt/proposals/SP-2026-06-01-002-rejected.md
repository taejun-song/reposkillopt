# Skill Edit Proposal SP-2026-06-01-002

## Proposed text

Replace the bounded secondary-structure additions from `SP-2026-06-01-001` with a single broader rule in **Repository Understanding Workflow** stage (a):

> Before producing the spec, recursively enumerate **every** subdirectory of **every** top-level directory and assign each a role. No directory is skipped.

+    Before producing the spec, recursively enumerate every subdirectory of every top-level directory and assign each a role. No directory is skipped.

## Justification

The supporting feedback (`FB-2026-05-31-002`, `FB-2026-06-01-004`) demonstrates that *secondary* structures get dropped — specifically non-pytest test trees and internal modules. This proposal would generalize that to "enumerate everything everywhere."

**This proposal is rejected and preserved as part of the audit trail (FR-021).** The generalization is not supported by the evidence: the feedback is about two specific, high-value secondary structures, not about exhaustive recursion. A blanket mandate makes the workflow unbounded on large or monorepo codebases and directly conflicts with the skill's existing discipline to spend evidence budget where it matters and *not* overclaim from shallow-but-wide reads. The bounded `SP-2026-06-01-001` is the right response; this one is kept on disk with `status: rejected` so the reasoning is not lost.
